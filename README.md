# Чашка кофе

Новый сайт сети ресторанов «Чашка кофе», который заменит `chashkacoffee.ru`.

Публичные страницы и меню разрабатываются как лёгкий SEO-ориентированный сайт; операционный контент, заявки и доступы сотрудников — как отдельная админка. Полное продуктовое описание находится в [техническом задании](./chashka-coffee-technical-spec.md).

## Принятые решения

- **Slug:** `chashka-coffee`.
- **Активные поверхности:** `website` (Astro), `webapp` (React-админка), `backend` (Bun/Hono API) и `packages/contracts` (общие DTO и валидация).
- **Отложено:** мобильное приложение; разработка ведётся на основной ветке шаблона без Expo.
- **Первый пользовательский маршрут:** список ресторанов → страница ресторана → выбор и просмотр меню.
- **Данные:** PostgreSQL локально через Docker Compose; сотрудникам понадобится авторизация и роли. Публичной пользовательской авторизации нет.
- **Файлы и интеграции:** медиатека и изображения предусмотрены в архитектуре; до появления сервера используются локальные файлы. Telegram понадобится для уведомлений о заявках позднее.
- **Не входит сейчас:** платежи, real-time/WebSocket-функции, домен, сервер и деплой.

## Структура

- `website` — публичный Astro-сайт, индексируемые страницы и SEO.
- `webapp` — закрытая React-админка для сотрудников.
- `backend` — монолитный Bun/Hono API, Prisma и фоновые задачи.
- `packages/contracts` — общие Zod-схемы и TypeScript-типы.
- `docs` — архитектурные, локальные и эксплуатационные заметки.

## Локальный запуск

Проект использует Bun `1.3.14` и Docker Desktop с Compose v2. Bun пока должен быть установлен на рабочем компьютере; PostgreSQL нативно не нужен.

```bash
bun install
docker compose up -d postgres
cp backend/.env.example backend/.env
bun run --cwd backend prisma:migrate
```

В отдельных терминалах:

```bash
bun run dev:backend
bun run dev:webapp
bun run dev:website
```

Подробности по локальной БД: [docs/LOCAL_DATABASE.md](./docs/LOCAL_DATABASE.md). До подключения деплоя проект работает только локально.
